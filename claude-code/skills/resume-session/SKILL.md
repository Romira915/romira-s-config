# Resume Session

## Instructions

前回のセッションの続きから作業を再開するために、現在の状況を把握・報告する。

1. メモリファイル (`MEMORY.md`) を読み、プロジェクトの状況を把握する
   - 詳細ファイルへのリンクがあれば必要に応じて読む
2. 現在のgit状態を確認する
   - `git branch` で現在のブランチ
   - `git status` で未コミットの変更
   - `git log --oneline -5` で直近のコミット
   - 未マージのPRがあれば `gh pr list` で確認
3. 以下のフォーマットでユーザーに報告する

```
## 現在の状況

### ブランチ: `{branch_name}`
{未コミット変更があれば記載}

### 直近の作業
- {完了したタスクを箇条書き}

### 進行中・残タスク
- [ ] {残っているタスクを箇条書き}

### 課題・ブロッカー
- {あれば記載、なければ「なし」}

何から始めますか？
```

## Notes

- 報告は簡潔に。詳細はユーザーに聞かれたら答える
- メモリに残タスクがあればそれを提示し、ユーザーに次のアクションを確認する
- PRのCIステータスやマージ状況が変わっている可能性があるので、最新状態を確認する
