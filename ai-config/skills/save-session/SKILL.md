# Save Session

## Instructions

現在のセッションの作業状況を MEMORY.md に簡潔に保存し、短時間中断後にコンテキスト共有少なめで再開できるようにする。

## 使い分け

- **save-session**: 軽量保存。今日中〜近いうちに自分で再開する、詳細な経緯は不要、MEMORY.md の更新だけで足りる場合。
- **handoff**: 詳細引き継ぎ。別日再開・別エージェント/別セッションに完全に渡す・意思決定や未解決事項を長めに残したい場合。

1. 今回のセッションで行った作業を振り返る
   - git log / git status / git diff で変更内容を確認
   - 会話中のタスク進捗を整理
2. 以下の情報をメモリファイル (`MEMORY.md`) に反映する
   - **完了したタスク**: PR番号、マージ状況、主な変更点
   - **進行中のタスク**: 現在のブランチ、PR状態、残作業
   - **課題・ブロッカー**: 未解決の問題、回避策
   - **残タスク**: 次にやるべきこと
   - **コード構成の変更**: 新規追加されたファイル、設定変更
3. `MEMORY.md` は200行以内に収める。詳細が必要な場合は別ファイルに分割しリンクする
4. 古い情報や完了済みで不要な情報は整理・削除する

## Notes

- メモリの保存先はプロジェクトのauto memoryディレクトリ
- 既存のメモリ内容を破壊しない。差分更新する
- 簡潔に書く。次の自分（別セッション）が5秒で状況把握できることを目指す
- 詳細な経緯、判断理由、次回冒頭に貼るプロンプトが必要なら `handoff` に切り替える
- 将来タスクのチェックボックスは完了したら `[x]` に更新するか削除する
