# CodeRabbit レビュー対応スキル

`coderabbit` CLIでコードレビューを実行し、各指摘の対応要否を判断して報告する。
ユーザーが承認した修正のみコードに適用する。

## 引数

- `$ARGUMENTS` にオプションを指定できる（例: `--base main`）
- 省略時はコミット済み変更を対象にレビューする

## 手順

### Step 1: CodeRabbit レビュー実行

コミット済みの変更に対してレビューを実行する：

```
coderabbit review -t committed --plain
```

引数でオプションが指定されている場合はそれを追加する（例: `--base <branch>`）。

出力が長い場合があるため、全文を取得すること。

### Step 2: 指摘の解析

CodeRabbitの出力から個別の指摘（finding）を抽出する。各指摘について：

1. **ファイルパスと行番号**を特定する
2. **指摘内容**（説明、重要度）を把握する
3. **該当ファイル・行**の現在のコードを読み、指摘が妥当か確認する

### Step 3: コメント分類・判断

各指摘について、以下の分類で対応要否を判断する：

| 分類 | 説明 | アクション |
|------|------|-----------|
| **対応する** | バグ、セキュリティ問題、明確な改善 | コード修正を提案 |
| **対応しない（理由あり）** | 設計意図と異なる、誤検知、些細 | スキップ理由を記載 |
| **要相談** | トレードオフがある、大きな変更が必要 | ユーザーに判断を委ねる |

#### 判断基準

- **対応すべき**: バグ修正、セキュリティ脆弱性、未処理のエラーケース、明確なロジックエラー
- **対応しない**: Nitpick（些細なスタイル指摘）でプロジェクト規約に反しないもの、誤検知（コードを読めば問題ないとわかるもの）
- **要相談**: パフォーマンス改善（トレードオフあり）、アーキテクチャ変更提案、スコープ外の改善提案

### Step 4: 結果レポート

以下のフォーマットで結果を報告する：

```
## CodeRabbit レビュー対応判断

レビュー対象: committed changes
指摘数: <N>件

---

### 対応する（<n>件）

#### 1. `<ファイルパス>` L<行番号>
- **指摘**: <指摘内容の要約>
- **判断**: <対応する理由>
- **修正方針**: <どう直すか>

...

### 対応しない（<n>件）

#### 1. `<ファイルパス>` L<行番号>
- **指摘**: <指摘内容の要約>
- **理由**: <対応しない理由>

...

### 要相談（<n>件）

#### 1. `<ファイルパス>` L<行番号>
- **指摘**: <指摘内容の要約>
- **論点**: <判断に必要な情報・トレードオフ>

---

対応を進めてよいですか？（個別に選択も可能です）
```

### Step 5: ユーザー承認後の修正

- ユーザーが「対応する」と承認した指摘についてのみ、コード修正を実施する
- **1指摘ずつ修正し、各修正後にユーザーに確認を取る**（フィードバックルールに従う）

## 注意事項

- このスキルはレビュー実行と判断レポートまでを自動で行い、実際のコード修正はユーザーの承認後にのみ実施する
- レポートではCodeRabbitの原文を要約し、冗長な出力は省略する
- `--agent` オプションは構造化出力が得られるが、`--plain` の方が読みやすいためデフォルトは `--plain` を使用する
- CLAUDE.md を追加の指示として渡したい場合は `-c CLAUDE.md` を付与できる
