# Handoff

## Instructions

現在のセッションの作業を整理し、次のセッションへ引き継げる状態にする。
セッション終了前に実行するスキル。

### 1. セッションの振り返り

現在のセッションで行った作業を整理する:
- 会話中のやり取りから作業内容・意思決定・未解決事項を抽出
- git管理プロジェクトの場合: `git status`, `git log`, `git diff` で変更内容を確認
- 進捗状況（完了/進行中/未着手）を整理

### 2. セッションドキュメントの作成

プロジェクトのauto memoryディレクトリ内に `sessions/` ディレクトリを作成（なければ）し、以下の形式でドキュメントを保存する:

**ファイル名:** `session_YYYY-MM-DD_<topic>.md`（topicは英語kebab-case、作業内容を端的に表す）

**フォーマット:**

```markdown
# Session: <作業の概要タイトル>

Date: YYYY-MM-DD

## やったこと
- 完了した作業を箇条書き

## 進行中
- 途中の作業と現在の状態
- どこまで終わっていて、次に何をすべきか

## 技術的な判断・メモ
- 採用したアプローチとその理由
- 試して不採用にしたアプローチとその理由
- ハマったポイントと解決策

## 残タスク
- [ ] 次にやるべきこと（優先度順）

## 関連ファイル
- 変更・作成したファイルのパス一覧
```

### 3. MEMORY.md の更新

- 既存のメモリ内容を壊さず差分更新する
- セッションに関連する既存エントリがあれば更新する
- 新しいセッションドキュメントへのリンクを追加する（不要になった古いセッションリンクは削除）
- 200行以内に収める

### 4. 次回プロンプトの表示

以下の形式で、次のセッションの冒頭で入力するプロンプトをユーザーに表示する:

```
---
次のセッションで以下を入力してください:

前回のセッション引き継ぎドキュメント `<セッションドキュメントのパス>` を読んで、続きの作業を始めて。

残タスク:
- <残タスク1>
- <残タスク2>
---
```

- 残タスクはセッションドキュメントの残タスクから主要なものを抜粋
- パスはプロジェクトのauto memoryディレクトリからの相対パスではなく、絶対パスで記載

## Notes

- このスキルはセッション終了の合図。実行後はユーザーが `/clear` するか新セッションを開始する想定
- `save-session` との違い: こちらは詳細ドキュメントを残し、次回プロンプトも提示する。`save-session` はMEMORY.mdの更新のみ
- セッションドキュメントは作業が完全に完了したら削除してよい（溜め込まない）
- 同日に複数セッションがある場合はtopicで区別する
